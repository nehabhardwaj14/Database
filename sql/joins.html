JOINS

As the name shows, JOIN means to combine something. In case of SQL, JOIN means "to combine two or more tables".

The SQL JOIN clause takes records from two or more tables in a database and combines it together.

TYPES OF JOINS

  • EQUI JOIN

  • LEFT JOIN

  • RIGHT JOIN

  • FULL JOIN

  • CARTETIAN JOIN

EQUI JOIN/INNER JOIN

  • Equi join returns the matching column values of the associated tables. It uses a comparison operator in the WHERE clause to refer equality.

            
SYNTAX:

            SELECT column_list FROM table1, table2....  
             WHERE table1.column_name =  table2.column_name; 
              
EXAMPLE:

              mysql>  SELECT * FROM employees INNER JOIN departments
               ON employees.department_id = departments.dept_id;

              +---------------------------------------------------------+ 
              | emp_id | emp_name | department_id | dept_id | dept_name |
              |--------|----------|---------------|---------|-----------|
              |   1    |   Alice  |       101     |   101   |   Sales   |
              |   2    |   Bob    |       102     |   102   |   IT      |
              |   3    |   Charlie|       101     |   101   |   Sales   |
              |   4    |   David  |       103     |   103   |   HR      |
              +---------------------------------------------------------+

LEFT JOIN

  • Returns all rows from the left table and the matched rows from the right table. If there are no matching rows in the right table, NULL values are returned for the columns of the right table.

            
SYNTAX:

            SELECT columns FROM table1 LEFT JOIN table2 ON table1.column = table2.column;
 
            
EXAMPLE :

            mysql> SELECT *FROM employees LEFT JOIN departments ON
             employees.department_id = departments.dept_id;
            +---------------------------------------------------------+
            | emp_id | emp_name | department_id | dept_id | dept_name |
            |--------|----------|---------------|---------|-----------|
            |   1    |   Alice  |       101     |   101   |   Sales   |
            |   2    |   Bob    |       102     |   102   |   IT      |
            |   3    |   Charlie|       101     |   101   |   Sales   |
            |   4    |   David  |       103     |   103   |   HR      |
            +---------------------------------------------------------+

RIGHT JOIN

  • Returns all rows from the right table and the matched rows from the left table. If there are no matching rows in the left table, NULL values are returned for the columns of the left table. 

            
SYNTAX:

            SELECT columns FROM table1 RIGHT JOIN table2 ON table1.column = table2.column;
            
            
EXAMPLE :

            mysql> SELECT *FROM employees RIGHT JOIN departments ON
             employees.department_id = departments.dept_id;
            +---------------------------------------------------------+
            | emp_id | emp_name | department_id | dept_id | dept_name |
            |--------|----------|---------------|---------|-----------|
            |   1    |   Alice  |       101     |   101   |   Sales   |
            |   2    |   Bob    |       102     |   102   |   IT      |
            |   3    |   Charlie|       101     |   101   |   Sales   |
            |   4    |   David  |       103     |   103   |   HR      |
            +---------------------------------------------------------+

FULL JOIN

  • Returns all rows when there is a match in either the left or right table. If there is no match, NULL values are returned for the columns of the table without a matching row.

            
SYNTAX:

            SELECT columns FROM table1 FULL JOIN table2 ON table1.column = table2.column;
            
            
EXAMPLE :

            mysql>  SELECT * FROM employees FULL JOIN departments 
            ON employees.department_id = departments.dept_id;
            +---------------------------------------------------------+
            | emp_id | emp_name | department_id | dept_id | dept_name |
            |--------|----------|---------------|---------|-----------|
            |   1    |   Alice  |       101     |   101   |   Sales   |
            |   2    |   Bob    |       102     |   102   |   IT      |
            |   3    |   Charlie|       101     |   101   |   Sales   |
            |   4    |   David  |       103     |   103   |   HR      |
            +---------------------------------------------------------+

CROSS JOIN

  • Returns the Cartesian product of the two tables, i.e., all possible combinations of rows. It does not require any condition to be met

            
SYNTAX:

             SELECT * FROM TABLE1_NAME CROSS JOIN TABLE2_NAME;
            
EXAMPLE:

            mysql> SELECT *FROM employees CROSS JOIN departments;
            +---------------------------------------------------------+
            | emp_id | emp_name | department_id | dept_id | dept_name |
            |--------|----------|---------------|---------|-----------|
            |   1    |   Alice  |       101     |   101   |   Sales   |
            |   2    |   Bob    |       102     |   102   |   IT      |
            |   3    |   Charlie|       101     |   103   |   HR      |
            |   4    |   David  |       103     |   101   |   Sales   |
            |   1    |   Alice  |       101     |   102   |   IT      |
            |   2    |   Bob    |       102     |   101   |   Sales   |
            |   3    |   Charlie|       101     |   102   |   IT      |
            |   4    |   David  |       103     |   102   |   IT      |
            +---------------------------------------------------------+
